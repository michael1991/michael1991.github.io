ORC File Format

1. 常见数据存储方式：

  • 1.1 水平的行存储（最常见）：

    • 一行中很多列，我们只取很少的几列，但是却需要把一行所有列读出来再筛选；
    • 一行中列的类型不一致，压缩比低；

  • 1.2 垂直的列存储：一行中的列可能分布在多个块上（不同行）；

  • 1.3 混合的PAX存储：结合行存储和列存储的优势，RCFile就继承于此思想；

2. RCFile

  • RowGroup为RCFile的基本单位

    • sync marker：标记连续RowGroup
    • metadata header：记录每个RowGroup有多少行数据，每列多少字节以及每列一行数据的字节数
    • data：实际数据，按列存储

  • 视每一列为无含义的二进制块

3. ORCFile

3.1 Stripe

每个stripe默认250MB，相当于rcfile的RowGroup，不过大小由4MB变为250MB，提升顺序读的吞吐率。每个Stripe里有三部分组成，分别是Index Data,Row Data,Stripe Footer；

Index Data

一个轻量级的index，默认是每隔1W行做一个索引。这里做的索引只是记录每行的各字段在Row Data中的offset以及每个Column的max和min值。

Row Data

存的是具体的数据，和RCfile一样，先取部分行，然后对这些行按列进行存储。与RCfile不同的地方在于对每个列进行了编码，分成多个Stream来存储，进一步减少所占空间。

Stripe Footer

存的是各个Stream的类型，长度等信息。

3.2 File Tail

PostScript

主要是用来存储压缩参数及压缩页脚的大小

Footer

每个文件有一个File Footer，存的是每个Stripe的行数，每个Column的数据类型信息等。

读取顺序

在读取文件时，会seek到文件尾部读PostScript，从里面解析到File Footer长度，再读FileFooter，从里面解析到各个Stripe信息，再读各个Stripe，即从后往前读。
